# Local Development Setup

How to run Margin IQ locally.

---

## Prerequisites

- Node.js (LTS recommended)
- npm
- Git

---

## Clone and Install

```bash
git clone <repo-url>
cd margin-maestro
npm install
```

---

## Environment Variables

Create `.env.local` at the project root with:

```
VITE_SUPABASE_URL=https://<project-id>.supabase.co
VITE_SUPABASE_PUBLISHABLE_KEY=<your-anon-key>
SUPABASE_URL=https://<project-id>.supabase.co
SUPABASE_PUBLISHABLE_KEY=<your-anon-key>
```

### Required

| Variable | Purpose |
|----------|---------|
| `VITE_SUPABASE_URL` | Supabase project URL (client-side, build-time) |
| `VITE_SUPABASE_PUBLISHABLE_KEY` | Supabase anon/publishable key (client-side, build-time) |
| `SUPABASE_URL` | Supabase project URL (server-side, runtime) |
| `SUPABASE_PUBLISHABLE_KEY` | Supabase anon/publishable key (server-side, runtime) |

### Forbidden

| Variable | Why |
|----------|-----|
| `VITE_SUPABASE_SERVICE_ROLE_KEY` | Never expose service role key to client. Use non-VITE prefix only. |

### Optional (server-side only, not needed for dev)

| Variable | Purpose |
|----------|---------|
| `SUPABASE_SERVICE_ROLE_KEY` | Server admin client (bypasses RLS). Only for server functions. |

## Supabase Key Setup

1. Go to your Supabase project dashboard.
2. Navigate to Settings → API.
3. Copy the **Project URL** → use as `VITE_SUPABASE_URL` and `SUPABASE_URL`.
4. Copy the **anon/public key** → use as `VITE_SUPABASE_PUBLISHABLE_KEY` and `SUPABASE_PUBLISHABLE_KEY`.
5. The **service_role key** is only needed for server-side admin operations and must NOT be prefixed with `VITE_`.

## Common Issues

### Missing VITE_SUPABASE_URL or VITE_SUPABASE_PUBLISHABLE_KEY
**Symptom:** App crashes on load with "Missing Supabase environment variable(s)."
**Fix:** Create `.env.local` with the required variables. See above.

### Wrong key name (VITE_SUPABASE_ANON_KEY)
**Symptom:** HTTP 500 or authentication errors.
**Fix:** This project uses `VITE_SUPABASE_PUBLISHABLE_KEY`, not `VITE_SUPABASE_ANON_KEY`. Check your `.env.local`.

### Session lost on refresh
**Symptom:** After login, refreshing the page loses the session.
**Status:** Known bug — Build 1.0E fix in progress. See `docs/auth-session-debug-plan.md`.

### Git line ending noise on Windows
**Symptom:** Large diffs showing only line ending changes.
**Fix:** Configure Git: `git config core.autocrlf true` or add a `.gitattributes` file.

### routeTree.gen.ts regeneration
**Symptom:** Route tree file shows as changed after dev server starts.
**Note:** This file is auto-generated by TanStack Router. Do not edit it manually. It regenerates when route files change.

## Important Notes

- **Never commit `.env.local`** — it contains project-specific secrets.
- **Never commit `.env`** with real credentials to a public repository.
- The build configuration is handled by `@lovable.dev/vite-tanstack-config` — do not add duplicate Vite plugins.
- The app uses TanStack Start (SSR). The dev server runs both server and client code.
